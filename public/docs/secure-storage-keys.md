# Secure Storage API Keys

This document explains how to secure your storage connector API keys (S3/DO Spaces access keys and secrets) using encryption.

## Why This Matters

By default, storage connector API keys are stored in plaintext in the database. If anyone gains access to your database, they could obtain your S3/DO Spaces credentials and access your storage.

This feature adds AES-256-GCM encryption to protect your API keys using a master key system.

## How It Works

The system uses a **master key** to encrypt/decrypt your storage API keys:

1. **Master Key**: A 256-bit cryptographic key used to encrypt/decrypt your storage API keys
2. **Encrypted Keys**: Your S3/DO Spaces API keys are stored encrypted in the database
3. **Automatic Decryption**: When connecting to storage, keys are automatically decrypted using the master key

## Setup Options

### Option 1: Environment Variable (Recommended for Production)

Set the master key as an environment variable:

```bash
# Add to your server configuration
PHUPPI_STORAGE_KEY_MASTER_KEY=your-generated-256-bit-key-here
```

### Option 2: Key File (Alternative for Development)

The key can be stored in a file at `data/storage-key.key`. This is useful if you cannot use environment variables.

## Setting Up Secure Keys

### Step 1: Navigate to Settings

Go to **Admin > Settings** in your Phuppi application.

### Step 2: Generate a Master Key

Click on the **Secure Keys** tab in the settings page.

Click the **Generate New Key** button. A new cryptographically secure 256-bit key will be generated.

### Step 3: Copy Your Key

**Important**: The key is shown only once! Copy it immediately to a safe place.

### Step 4: Configure the Key

Choose one of the following methods:

#### For Production (Environment Variable)

Add the key to your server's environment:
- Apache: `SetEnv PHUPPI_STORAGE_KEY_MASTER_KEY "your-key-here"`
- Nginx: `fastcgi_param PHUPPI_STORAGE_KEY_MASTER_KEY "your-key-here";`
- Docker: Add to your docker-compose.yml environment section

#### For Development (File)

When you click "Generate New Key", check the option to save to file, or use the "Save Key to File" button in the UI.

### Step 5: Restart Your Application

After setting the environment variable or key file, restart your PHP application for the changes to take effect.

## How Encryption Works

1. **Adding a New Connector**: When you add an S3/DO Spaces connector, the API keys are automatically encrypted if secure mode is available
2. **Updating a Connector**: When you update keys, they are re-encrypted
3. **Using a Connector**: Keys are decrypted on-the-fly when connecting to storage

## Security Considerations

### Key Display
- Generated keys are shown only once - there is no way to retrieve them later
- If you lose your key, you will need to generate a new one and update all connectors

### Key Storage
- Environment variables are more secure than files (files can be accidentally committed or exposed)
- Key files have restricted permissions (0600) to prevent unauthorized access

### Key Rotation
- If you regenerate the master key, existing encrypted API keys will become invalid
- You will need to update all S3/DO Spaces connectors with new credentials

## Troubleshooting

### "No master key configured" Warning

This appears when you have S3/DO Spaces connectors but no master key is set. Your API keys are stored in plaintext.

**Solution**: Generate a master key and configure it as described above.

### "Invalid key format" Error

The key stored in the environment variable or file is not in the correct format. Ensure it matches the format generated by the key wizard (64 hex characters).

### "Decryption failed" Error

This can happen if:
- The master key doesn't match the one used to encrypt the keys
- The encrypted data was corrupted

**Solution**: Check that your master key is correctly configured. If keys were encrypted with a different key, you may need to re-enter your API credentials in the connector settings.

## Checking Key Status

In the **Secure Keys** tab, you can see:
- **Green status**: Secure mode is active and keys are encrypted
- **Yellow status**: No master key configured - keys are stored in plaintext
- **Red warning**: S3/DO Spaces connectors exist without secure mode

## For Existing Installations

If you're upgrading from a version without this feature:

1. The existing plaintext keys will continue to work
2. A migration will automatically encrypt existing keys if a master key is configured
3. If no master key is available, keys remain plaintext with a warning in the UI
